HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Root element in which all other elements go inside of.
  • lang represents which language the page is in.

CSS

  • Determines the style of the page.
  • Padding-recommends the amount of space needed for content inside of an element.
  • Margin-recommends the amount of space needed for content outside of an element.
  • the head section is where metadata is stored which contains information about the webpage
  • rel specifies the relationship between the current document and the linked document resource
  • href species the location(URL) of the external resource

Git

  • Contains project files.
  • allows the user to manage aswell as discuss project work with others.
  • you dont always want be be on main so your partner can review what changes you have made before the main piece is changed.
  • git status-checks what branch the user is on.
  • git checkout -b branch-name:creates a new branch and switches to it.
  • Important to frequently sync local version of your code with the remote version while in github.
  • If the local version is not frequently updated it can lead to big sync problems.
  • git add -A commits all files in the current branch.
  • git commit -m "added code to HTML File" commits the changes to the respository.
  • -m associates a message with the commit.

JavaScript

  • Case sensitive.
  • Variable names cannot start with a number.
  • No value is needed when the variable is declared.
  • Used for websites aswell as applications.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
